# Summary

Implemented Bridge Status Indicator as two files:

## Files Created

### `lib/mc/use-bridge-status.ts`

- Polls `{baseUrl}/api/openclaw/status` every 10 seconds
- Returns `{ connected: boolean, loading: boolean }`
- Uses useEffect with interval cleanup and active flag to prevent state updates after unmount
- Fetch failures set connected=false (graceful degradation)

### `components/mc/bridge-status-indicator.tsx`

- Accepts `{ baseUrl: string }` props
- Renders inline-flex dot + "Bridge" label for header placement
- Green (bg-emerald-400) when connected, red (bg-red-400) when disconnected
- Animated pulse on zinc-500 during initial loading state
- Tailwind CSS only, no external dependencies

## Patterns Followed

- "use client" directive matching existing hooks (use-mc-websocket.ts)
- Path alias `@/lib/mc/` for imports matching project conventions
- Consistent Tailwind color palette (zinc, emerald) with existing components
