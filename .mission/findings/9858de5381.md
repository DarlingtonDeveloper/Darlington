# Summary

Implemented FindingsView component and wired it into the MC dashboard trace view.

## Changes

### New: `components/mc/findings-view.tsx`

- Tabbed UI with **Findings** (markdown) and **Briefing** (JSON) tabs
- Props: `taskId: string | null`, `baseUrl: string`
- Fetches `/api/tasks/{id}/findings` (markdown) and `/api/tasks/{id}/briefing` (JSON) in parallel
- Uses `react-markdown` + `remark-gfm` for rich markdown rendering
- Full dark theme styling: bg `#07070e`, accent `#c4b5a0`, DM Sans for prose, JetBrains Mono for code
- Custom components for all markdown elements (headings, code blocks, tables, blockquotes, links)
- Handles loading, error, and empty states
- Auto-selects the tab that has data (falls back to findings)

### Modified: `app/mc/mc-client.tsx`

- Added `selectedTaskId` state lifted to MCClient
- Passes `selectedTaskId` and `onSelectTask` to TraceView
- Renders FindingsView below TraceView when a task is selected

### Modified: `components/mc/trace-view.tsx`

- Added optional `selectedTaskId` and `onSelectTask` props
- Backwards-compatible: falls back to internal state when props not provided
- Clicking a node in the graph now propagates selection to parent

### Dependencies: `package.json`

- Added `react-markdown@^10.1.0` and `remark-gfm@^4.0.1`

Compilation successful. Pre-existing Supabase env error on `/api/health/webhook/evening` is unrelated.
