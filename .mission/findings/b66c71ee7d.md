## Implementation

- **File:** `components/mc/worker-panel.tsx`
- **Directive:** `"use client"` at top
- **Export:** `export default function WorkerPanel`
- **Props:** `{ workers: Worker[] }` with `Worker` imported from `@/lib/mc/types`

## Features

- Lists workers showing: persona badge (text-[#c4b5a0]), truncated task ID, token count (text-zinc-400), status badge
- Status badge colors: spawning=warning/yellow, running/busy=accent/blue, complete/idle=success/green, failed/error=danger/red, offline=default
- Click-to-expand rows revealing: full task ID, session ID, persona, zone, uptime, tokens, cost
- Footer with total and active worker counts using Badge components
- Empty state when no workers present

## Design Decisions

- Followed existing `mission-view.tsx` patterns: Badge component usage, border-white/[0.04] borders, bg-black/15 row backgrounds, font-mono throughout
- Matched dark theme palette from design spec: text-[#c4b5a0] accents, text-[#6b6560] labels, text-[#4a4540] muted, text-[#a89880] values
- Used `useState<string | null>` for accordion expand (single row at a time)
- Tailwind CSS only, no inline styles
- Helper functions `formatTokens`, `formatUptime`, `truncateId` defined locally (can be extracted to shared utils in T6)
